The Illusion of AI Thinking

Report on “The Illusion of AI Thinking”​

by Apple ML Research, June 2025

1. Context & Motivation​

Recent “large reasoning models” (LRMs) expose an optional thinking/synching mode in which the model emits internal chain-of-thought tokens. Public leaderboard results (ARC-AGI, MATH500, etc.) show large jumps in accuracy when this mode is enabled, so many practitioners assume LRMs genuinely reason in a human-like way. Apple’s paper questions that assumption and asks:

Are LRMs truly reasoning, or merely creating a convincing illusion of reasoning?

2. Experimental Setup​

AspectDetails
ModelsClaude 3.7 (non-sync vs. sync), Claude Sonnet 4, DeepSeek-R1, Gemini 2.5 preview, etc.
TasksFour classical algorithmic puzzles chosen for precise complexity control and verifiable solutions: 1) Tower of Hanoi, 2) Checker jumping, 3) River-crossing, 4) Block-world rearrangement.
Complexity AxisNumber of objects/disks to move (1 → 12). Defines three regimes: Low, Medium, High complexity.
MetricsAccuracy, number of reasoning tokens, wall-clock latency.
Interventions1. Enable/disable thinking mode; 2. Vary thinking-token budget (8 k → 32 k); 3. Provide step-by-step solution templates (few-shot / RL fine-tuning).

3. Key Findings​

  1. Three-phase performance curve

    • Low complexity (≤ 2 objects)
      non-thinking baseline already solves tasks; thinking adds no value and is sometimes slightly worse.

    • Medium complexity (≈ 4 – 8 objects)
      thinking mode strongly outperforms; more tokens → higher success.

    • High complexity (≥ 10 objects)
      both modes collapse to ~0 accuracy
      Surprisingly, the model emits fewer thinking tokens beyond this point; evidence it “gives up” instead of trying harder.

  2. Token-count inversion

    • Expectations: harder task → longer chain-of-thought.

    • Observation: after a sharp threshold, token length drops even as complexity rises.
      This contradicts the view that LRMs search deeper when challenged.

  3. Ineffectiveness of additional guidance Providing explicit solution templates or RL fine-tuning did not move the collapse point. Models still failed at the same complexity tier.

  4. Memorisation vs. reasoning Claude 3.7 could execute ~100 flawless moves on the well-studied Tower of Hanoi but failed after < 5 moves on an obscure river-crossing variant requiring only 11 total moves, indicating recall of memorised patterns, not general reasoning.

  5. Tool / code execution sidesteps the issue When the LRM delegates planning to external code (Python/C++ solver) it instantly produces perfect solutions, highlighting that the failure lies in transformer-based reasoning rather than the tasks themselves.

4. Interpretation​

  • Illusion Thesis
    The apparent “reasoning” bump at medium complexity is fragile and narrow. Beyond a modest threshold, LRMs abandon systematic search and revert to shorter, often hallucinatory outputs.

  • Pattern-matching Dominance
    Success on famous benchmarks stems largely from memorised algorithmic templates seen in pre-training data; unfamiliar but isomorphic tasks expose the weakness.

  • Scaling Limits
    Merely increasing context window (8 k → 32 k) or parameter count does not extend the reasoning frontier.

  • Reinforcement Learning Limits
    RL fine-tuning improves style adherence and detox, but doesn’t endow models with fundamentally deeper deductive ability.

  • Practical Implication
    For workloads ≈ medium complexity (e.g. 2-to-3-step logical problems) thinking mode is worthwhile; for truly intricate algorithmic planning, direct code execution or specialised symbolic solvers remain essential.

5. Implications for Practitioners​

  1. Benchmark responsibly
    Relying on saturated public leaderboards can mislead; craft domain-specific, unpublished tests to measure genuine reasoning.

  2. Budget thinking tokens prudently
    Enable thinking when tasks sit in the “medium” band; disable it for trivial queries to save latency & cost.

  3. Augment with tools
    For complex combinatorial planning, integrate external solvers rather than expecting pure-LLM deduction.

  4. Monitor token-usage curves
    A sudden drop in reasoning-token length is a red-flag that the model has silently abandoned the problem.

6. Future Directions Suggested by Apple​

  • Architectural research into mechanisms that maintain or adaptively expand reasoning depth instead of collapsing.
  • Curriculum and data targeting under-represented algorithmic patterns.
  • Transparency: expose chain-of-thought for proprietary models to allow finer-grained diagnosis.

Bottom Line​

Apple’s “Illusion of AI Thinking” delivers a sobering reality-check: today’s LRMs exhibit genuine-looking reasoning only in a narrow Goldilocks zone of problem complexity. Beyond that, they default to short, often wrong outputs rather than “trying harder.” Effective deployment therefore requires calibrated use of thinking budgets, bespoke benchmarks, and—crucially—hybrid systems that delegate heavy logical lifting to explicit code or symbolic engines.
