7-Eleven’s Agentic Marketing Assistant

Post-mortem: what worked, what didn’t, and the tech stack that enabled it

1. Evolution at a glance​

PhaseGoalKey choicesOutcome
Early RAG chatbots (2023)Answer internal FAQs from PDFs & troubleshooting docsLlama-1 / Falcon, FAISS, on-prem quantised modelsGood for short docs; collapsed on 60-page finance PDFs → shelved
Summit “brick-bot” (2024)Conference Q&A in MS TeamsStructured data in Genie table instead of vectors~600 unique users, but hard-coded flows, no reasoning
Creatives Assistant v1 (poc)Campaign ideation, copy, general Q&ASupervisor + 3 sub-bots, Streamlit UIDemonstrated value; latency high; no reflection or tools
Creatives Assistant v2 (production)End-to-end campaign generationLangGraph multi-agent graph, web-search & email tools, guardrails, human-in-loopCuts hours of manual prep; preferred over ChatGPT by marketers

2. What worked​

CategoryPractices / components that paid off
Prompt & data designVerbose system prompts (50 lines) specified tone, brand voice, demographic, CTAs → fewer hallucinations. Concise context per agent: supervisor passes only the fields each sub-agent needs → higher precision.
Agent orchestrationSupervisor / sub-agent split with clear roles prevented context drift; reflection step let the supervisor restart internally when quality checks failed. Web-search agent stopped duplication of existing tag-lines and injected trend awareness.
Tooling & governanceLayered guardrails: custom toxicity/PII filter at the perimeter and Databricks AI Gateway inside → caught bad input & bad output. Human-in-loop checkpoints (LangGraph reviewer nodes) boosted trust without blocking the whole flow.
UX & adoptionStreamlit prototype + MS Teams integration meant one-click access; dropdown “config” panel let non-tech users set tone, demographic, CTA without prompt engineering.
ObservabilityMLflow trace logging + Databricks inference tables → prompt tweaks could be A/B-scored on groundedness, relevance, latency in minutes.
Performance gainsMarketing team now uploads a campaign brief and receives: 3 fully-formed concepts, multi-channel copy sets, execution summary -> hours of manual composition removed and more ideas explored.

3. What didn’t work (and why)​

SymptomRoot causeFix / lesson
Failing on large finance docsNaïve PDF OCR + equal-size chunking; retrieval good, answers poorPre-process: summarise tables, store semantic chunks only.
High latency in v1Sequential, uni-directional steps; no early exitsAdded reflection & branch-back, moved guardrail classifiers earlier, parallelised tools.
“Reasoning” tokens confused usersExposing raw chain-of-thought from reasoning modelsSwitched to short explanatory summaries rather than full CoT.
Sub-agents hallucinating competitors’ slogansFull context dump passed downstreamPass only campaign brief + web summary needed for the next task.
Initial guardrails bypassed via hostile promptsPrompt-only guardrails weakCombined model-level filters (AI Gateway) with open-source toxic/PII classifier API at entry & exit.

LayerTools / frameworksNotes

Agent frameworkLangGraphGraph DSL, tool decorators, stateful memory, human-in-loop nodes.
Vector & structured retrievalFAISS; Databricks Genie tablesGenie table for conference bot; FAISS for creative briefs.

Monitoring / evalMLflow traces, Databricks agent-evaluation (LLM-as-judge), inference tablesAutomated tests for groundedness, safety, relevance; CLI for prompt regression.

5. Key take-aways for similar projects​

  1. Separation of duties beats monolithic prompts. Keep supervisor thin and delegate specialised tasks to sub-agents with minimal context.
  2. Reflection checkpoints (plan → act → self-review) raise answer quality without user retries.
  3. Long, explicit system prompts are OK—clarity > brevity when you control latency.
  4. Tool integration (search, email, OCR, data-lake queries) turns a chatbot into an action model; start simple but design for pluggability.
  5. Guardrails must surround the entire graph, not just the foundation model.
  6. Observability early: trace every step, version every prompt, and automate basic eval so iterations stay data-driven.
  7. Adoption hinges on UX: embed where users already live (Teams) and surface adjustable levers instead of raw prompts.

Bottom line: the project succeeded once the team moved from a single-step RAG chatbot to a graph of specialised, tool-augmented agents with rigorous guardrails and observability. Technologies that enabled the leap—LangGraph, Databricks/Mosaic AI, Streamlit, and targeted open-source libraries—are all available off-the-shelf, but the win came from careful prompt/data design and tight alignment with marketing workflows.
